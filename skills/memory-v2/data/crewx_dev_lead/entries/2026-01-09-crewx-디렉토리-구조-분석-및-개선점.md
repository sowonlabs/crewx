# .crewx 디렉토리 구조 분석 및 개선점

## 핵심 발견

### 1. traces.db ↔ conversations 연결 없음
- task_id와 threadId가 서로 다른 ID 체계
- 특정 task가 어느 conversation에서 실행됐는지 역추적 불가
- 해결: tasks 테이블에 thread_id 컬럼 추가 필요

### 2. 데이터 중복 (3곳)
- traces.db: prompt, result
- conversations/*.json: messages
- logs/*.log: 전체 로그
- 총 47MB (conversations 4MB + traces 11MB + logs 32MB)

### 3. Cleanup 정책 미작동
- TracingService.cleanupOldTasks(30일) - 있지만 자동 호출 안됨
- ConversationStorageService.cleanupOldThreads(30일) - 있지만 자동 호출 안됨
- logs/ - cleanup 로직 자체 없음 (32MB, 577개 파일 누적)

## 개선 권고

1. 단기: crewx cleanup CLI 명령어 추가
2. 중기: task-conversation 연결 (thread_id 필드)
3. 장기: logs 파일을 traces.db logs 컬럼으로 통합 (Phase 4 확장)
